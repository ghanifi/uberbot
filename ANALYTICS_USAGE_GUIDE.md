# Analytics Dashboard - Usage Guide

## Accessing the Analytics Dashboard

1. Log in to WordPress Admin
2. Go to **Settings** > **Customer Analytics**
3. Dashboard loads with metrics for the last 30 days by default

## Dashboard Sections Explained

### 1. Overview Cards (Top Section)
Four key metric cards displaying:
- **Total Bookings**: Count of all bookings in selected period
- **Total Revenue (GBP)**: Sum of all booking values converted to GBP
- **Average Booking Value**: Total revenue divided by number of bookings
- **Top Source Site**: Regional site with most bookings in period

### 2. Time Period Filters
Located below the page title:
- **Time Period Dropdown**: Select from preset periods
  - Today
  - Last 7 Days
  - Last 30 Days (default)
  - This Month
  - This Year
  - Custom Range (shows date pickers)
  
- **Regional Site Dropdown**: Filter to single site or "All Sites"
- **Apply Filters Button**: Updates all metrics and charts

### 3. Charts Section (4 Visualizations)
Each chart updates based on selected filters:

**Bookings by Regional Site** (Doughnut Chart)
- Shows distribution of bookings across all regional sites
- Hover to see percentages
- Use to identify which regional sites are most active

**Revenue by Regional Site** (Horizontal Bar Chart)
- Displays total revenue per site in GBP
- Longest bar = highest revenue
- Use to identify most profitable regions

**Bookings by Language** (Pie Chart)
- Shows customer language preferences
- Each slice = one language
- Identifies which language markets are largest

**Daily Booking Trend** (Line Chart)
- Shows booking count for each day in last 30 days
- Peaks indicate high-volume periods
- Use to identify seasonal patterns

### 4. Regional Site Performance Table
Comprehensive breakdown of all regional sites:

| Column | Meaning |
|--------|---------|
| Site ID | Name of regional site (antalya, london, etc.) |
| Bookings | Number of bookings from this site |
| Revenue (GBP) | Total revenue generated by this site |
| Avg Value | Average booking value (Revenue / Bookings) |
| % of Total | Site's percentage of total revenue |

**How to use:**
- Compare revenue between sites
- Identify underperforming regions
- Track growth/decline over time

### 5. Language Breakdown Table
Analysis of bookings by customer language:

| Column | Meaning |
|--------|---------|
| Language | ISO 639-1 language code (EN, TR, DE, etc.) |
| Bookings | Number of bookings in this language |
| Revenue (GBP) | Total revenue from this language group |
| % of Total | Language's percentage of total bookings |

**How to use:**
- Identify which languages drive most business
- Plan marketing campaigns by language
- Allocate support resources

### 6. Recent Bookings Table
Detailed listing of individual bookings (20 per page):

| Column | Meaning |
|--------|---------|
| Booking ID | Link to full booking details in Reservations |
| Customer | Customer name (sortable) |
| Site | Which regional site the booking came from |
| Language | Customer's selected language |
| Date | When the booking was created |
| Amount | Price in original currency |
| Status | Current booking status (color-coded) |

**Status Colors:**
- Green = Completed ✓
- Yellow = Pending ⏳
- Red = Cancelled ✗

**How to use:**
- Review recent bookings
- Click Booking ID to view full details and edit
- Use pagination to navigate through bookings

## Using Filters

### Date Range Filters

**Quick Periods:**
- **Today**: Bookings created in last 24 hours
- **Last 7 Days**: Last 168 hours (Mon-Sun)
- **Last 30 Days**: Last 30 days regardless of month boundaries
- **This Month**: Calendar month to today
- **This Year**: January 1 to today

**Custom Range:**
1. Select "Custom Range" from dropdown
2. Date picker fields appear
3. Choose "From" and "To" dates (inclusive)
4. Click "Apply Filters"

### Regional Site Filter

- Select "All Sites" to see combined metrics
- Select specific site to isolate that region's performance
- All charts and tables update to show only selected site

## Exporting Data

### CSV Export

1. Scroll to "Recent Bookings" section
2. Click **"Export to CSV"** button
3. Browser downloads: `analytics_export_[timestamp].csv`

**CSV Contents:**
- Header row: Booking ID, Customer Name, Email, Site, Language, Transfer Date, Amount, Currency, Status
- One row per booking matching current filters
- Can open in Excel, Google Sheets, or any spreadsheet app

**Use Cases:**
- Share data with accounting/finance team
- Create custom reports
- Import to external analytics tools
- Backup booking data

## Interpreting Metrics

### Revenue Analysis
- **Total Revenue**: All bookings summed and converted to GBP
- **Conversion Applied**: If booking was in TRY or EUR, used current exchange rates
- **Should Match**: Reports from payment processor (Stripe) after accounting for pending/failed bookings

### Booking Trends
- **Upward Trend**: Increasing customer interest
- **Downward Trend**: May indicate issues or seasonality
- **Spikes**: Look for corresponding marketing campaigns or events

### Regional Performance
- **Consistent Leaders**: Regional sites with steady high revenue
- **Emerging Markets**: Sites showing rapid growth
- **Underperformers**: Sites with low bookings - consider targeted marketing

### Language Insights
- **Most Common**: Major language groups in customer base
- **Growth Languages**: Languages with increasing bookings
- **Underserved**: Languages with low usage despite availability

## Common Tasks

### Task: Find Top Regional Site
1. Look at Regional Site Performance table
2. Check "% of Total" column - highest percentage wins
3. Or look at Revenue (GBP) column for most revenue

### Task: Compare Two Months
1. Set filters to "This Month" - note metrics
2. Change to previous month using custom date range
3. Compare key metrics side-by-side

### Task: Export Q1 Data
1. Set date range: Jan 1 to Mar 31
2. Select "All Sites"
3. Scroll to Recent Bookings
4. Click "Export to CSV"
5. Opens in Excel with Q1 data

### Task: Analyze Seasonal Pattern
1. Select "Last 7 Days"
2. Look at "Daily Booking Trend" chart
3. Note which days have spikes
4. Look for day-of-week patterns

### Task: Identify Growth Opportunity
1. Review "Regional Site Performance" table
2. Look for sites with low current volume
3. Note their "Avg Value" - is it high?
4. Consider if low volume is marketing/awareness issue

## Troubleshooting

### Dashboard Not Loading
- Refresh page (F5)
- Clear browser cache
- Check WordPress admin is accessible
- Verify you have "manage_options" capability

### Charts Not Displaying
- Check browser console (F12) for JavaScript errors
- Ensure JavaScript is enabled
- Try different browser
- Check CDN is accessible: https://cdn.jsdelivr.net/npm/chart.js@3.9.1/

### Data Looks Wrong
- Verify filters are correct
- Check date range is inclusive of expected bookings
- Ensure regional site is correctly selected
- Look for bookings in Reservations > Reservations page

### Export Not Working
- Check popup blocker isn't blocking download
- Verify manage_options permission
- Try Firefox if Chrome has issues
- File should download to Downloads folder

## Performance Notes

- Dashboard loads metric data on page load (~1-2 seconds)
- Charts render after page load completes
- Large date ranges (1+ year) may take longer
- CSV export with 10,000+ bookings takes 5-10 seconds

## Security Notes

- Analytics dashboard only visible to administrators (manage_options capability)
- All AJAX requests verified with WordPress nonce tokens
- No sensitive customer data in default views
- Email addresses only shown when exporting CSV (with nonce verification)

## Database Queries Run

When dashboard loads:
1. WP_Query for bookings in date range
2. Meta queries for source_site filtering (indexed)
3. Meta queries for source_language
4. Exchange rate lookups (cached in options)
5. Pagination counts

Expected performance:
- <100 bookings: <100ms
- 1,000 bookings: <500ms
- 10,000 bookings: 1-2 seconds
